# 产品描述
- 为前端提供 RESTful API 接口，实现前端与后端的通信和功能交互

# 技术栈需求描述
- 基于 Node.js 的后端服务，提供 RESTful API 接口，所有接口都使用`POST`方法
- 使用fastify(https://github.com/fastify/fastify) 框架构建
- 使用TypseScript 进行开发
- 使用SQLite 作为数据库
- 工程结构清晰，便于维护和扩展
- 使用pnpm进行包管理
- 集成Swagger进行API文档生成

# 功能需求描述
- 支持用户认证和授权，使用 JWT 进行身份验证
- 提供 CRUD（创建、读取、更新、删除）操作的 API 接口
- 实现分页和过滤功能，提升数据查询效率
- 提供日志记录和错误处理机制，确保系统稳定运行
- 支持跨域资源共享（CORS），允许前端应用访问后端 API
- 支持文件上传和下载功能，实现文件管理
- 支持对接第三方服务，使用 RESTful API 进行数据交换
- 支持定时任务调度，执行周期性任务

# 非功能需求描述
- 接口标准按照OpenAPI规范编写，确保一致性和可维护性
- 系统应具备高可用性和可扩展性，支持未来功能扩展
- 提供详细的日志记录，便于问题排查和性能监控
- 确保数据安全，防止 SQL 注入和跨站脚本攻击
- 提供备份和恢复机制，确保数据完整性
- 系统应具备良好的性能，响应时间应在合理范围内
- 提供 API 文档，集成 Swagger，方便前端开发人员使用
- 支持多语言国际化，满足不同地区用户需求
- 提供自动化测试，确保代码质量和功能正确性
- 支持容器化部署，使用 Docker 提升部署效率


## Vibe记录
- @types/common.ts 中定义了通用接口的结构，其中`Response`是所有接口的统一应答结构,其他接口应答结构都是继承此结构，并且必须采用`{data: <sub-structure>}`格式定义。`Response`结构中`code`是`number`类型，并且没有`error`字段。`code`字段用于指示业务功能的结果，`0`代表成功，`-1`或其他负数代表失败。调整`backend`应用中的应答代码，如果`code`使用了`string`类型，可以将其定义为一个number常量，便于理解。

# 页面功能描述
- 页面主要用于引导用户创建应用
- 用户在页面中输入应用信息后，点击“创建应用”按钮，系统将根据用户输入的信息通过`Chat`方式多次交互后，创建一个新的应用实例
- 页面功能需要与后台服务通过API接口进行交互，实现应用功能逻辑
- 具体步骤如下：
 1. 用户在页面中输入应用需求描述
 2. 用户点击“创建应用”按钮，页面将收集用户输入的信息，并通过API接口将信息发送到后台服务
 3. 后台服务接收到请求后，开始与用户进行多轮`Chat`交互，收集更多的应用需求和细节信息
 4. 在多轮交互完成后，后台服务根据收集到的信息创建一个新的应用实例，并将创建结果返回给页面
 5. 页面接收到创建结果后，向用户展示应用创建成功的消息，并提供后续操作选项，如查看应用详情、继续创建新应用等
- 交互流程描述如下：
1. 初始状态下，页面上方为一个输入框和“创建应用”按钮，下方为现有应用列表，支持列表和卡片方式展示
2. 用户在输入框中输入应用需求描述，点击“创建应用”按钮，此时隐藏下方应用列表，将输入框移动到底部，上方开始展示`Chat`交互内容，直至交互结束

# 页面布局描述
- 页面分为上下两部分，上半部分为工作空间信息录入区，下半部分为工作空间应用列表。
## 上半部分布局
- 包含一个多行文本输入框和一个“创建应用”按钮
- 文本输入框用于用户输入工作空间的描述信息，支持多行文本输入
- “创建应用”按钮位于文本输入框下方，点击后触发创建应用的操作
## 下半部分布局
- 包含一个应用列表区域，用于展示当前工作空间下的所有应用
- 应用列表支持两种展示方式：列表视图和卡片视图，用户可以通过切换按钮选择不同的视图方式
- 列表视图以表格形式展示应用的基本信息，如应用名称、创建时间等
- 卡片视图以卡片形式展示应用的详细信息，如应用图标、描述等
- 应用列表区域支持分页显示，当应用数量较多时，用户可以通过分页控件浏览更多应用
